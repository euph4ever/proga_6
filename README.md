
​ВАРИАНТ 48: Электросамокаты
============================================================
Описание: Показания телематики арендного микротранспорта: заряд, пробег, скорость, разблокировки. Управление парком и техобслуживанием.

 Поля:
  • ts (int32): время замера
  • scoot_id (int16): ID самоката
  • batt (float32): заряд батареи, %
  • km (float32): пробег, км
  • sp (float32): скорость, км/ч
  • unlocks (uint16): количество разблокировок

Очистка: batt ∉ [0, 100] → clip. Заменить sp < 0 на 0. Заменить unlocks < 0 на 0.
Пояснение: Заряд ограничен 0–100%. Скорость и разблокировки не бывают отрицательными → замените на 0.

Группировка: По scoot_id. Средний km, макс sp.
Пояснение: Для каждого самоката посчитайте средний пробег и максимальную скорость. Это покажет интенсивность аренды и соблюдение скоростных ограничений.

Окно: k=35. Moving avg km. np.diff(batt) → разряд за поездку.
Пояснение: Сглаженный пробег выявит тренды использования. Изменения заряда между замерами укажут на энергопотребление и необходимость подзарядки.

Акцент: Суточный сброс одометра имитируется, но clip защищает.
Пояснение: При обслуживании пробег может обнуляться. Ограничивайте резкие падения верхними пределами, чтобы не ломать статистику агрегации.
